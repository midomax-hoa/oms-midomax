 lấy được dữ liệu đơn hàng và sản phẩm từ sàn Shopee về website (hệ thống oms.midomax.vn), bạn cần thực hiện theo quy trình kỹ thuật của Shopee Open API v2. Dưới đây là các bước chi tiết:1. Quy trình Ủy quyền (Authorization)Để app của bạn có quyền truy cập dữ liệu của một Shop cụ thể, bạn phải thực hiện bước "Authorize" để lấy access_token.Tạo Link ủy quyền: Bạn cần tạo một URL để chủ Shop đăng nhập và xác nhận cho phép App truy cập.Link mẫu: https://partner.shopeemobile.com/api/v1/shop/auth_partner?partner_id={partner_id}&timestamp={timestamp}&sign={sign}&redirect={redirect_url}Redirect URL: Phải trùng với URL bạn đã đăng ký trong ảnh: https://oms.midomax.vn/.Lấy Code: Sau khi chủ shop đồng ý, Shopee sẽ trả về một code qua URL redirect.Đổi Code lấy Token: Gọi API v2.auth.get_access_token để lấy access_token và refresh_token.Lưu ý: access_token thường có hạn trong 4 giờ, refresh_token có hạn trong 30 ngày.2. Cách ký tên (Signature Generation)Mọi yêu cầu API lên Shopee đều bắt buộc phải có tham số sign. Công thức tính toán (HMAC-SHA256) như sau:$$sign = hmac\_sha256(partner\_key, partner\_id + path + timestamp + access\_token + shop\_id)$$Partner ID: 2033121 (lấy từ ảnh của bạn).Partner Key: Bạn cần click vào biểu tượng "mắt" ở phần Live API Partner Key trong ảnh thứ 2 để lấy mã bí mật này.3. Lấy danh sách Sản phẩm (Products)Để đồng bộ sản phẩm về web, bạn sử dụng các Endpoint sau:Bước 1: Lấy danh sách ID sản phẩm: Dùng API v2.product.get_item_list.Tham số quan trọng: offset, page_size, item_status (NORMAL, BANNED, v.v.).Bước 2: Lấy chi tiết từng sản phẩm: Dùng API v2.product.get_item_base_info.Truyền vào danh sách item_id_list thu được ở Bước 1 để lấy tên, giá, mô tả và hình ảnh.4. Lấy danh sách Đơn hàng (Orders)Để quản lý đơn hàng trên CRM, bạn thực hiện:Lấy danh sách đơn hàng mới: Dùng API v2.order.get_order_list.Bạn có thể lọc theo time_range_field (ví dụ: create_time) để lấy các đơn hàng phát sinh trong khoảng thời gian nhất định.Lấy chi tiết đơn hàng: Dùng API v2.order.get_order_detail.API này trả về thông tin người mua, địa chỉ (đã mã hóa theo chính sách bảo mật), danh sách sản phẩm trong đơn, tổng tiền, trạng thái vận chuyển.5. Tự động hóa với Push Mechanism (Khuyên dùng)Thay vì phải liên tục gọi API để kiểm tra xem có đơn mới hay không (Polling), bạn nên sử dụng tính năng Push Mechanism (thấy ở menu bên trái trong ảnh 1).Cài đặt: Cấu hình một Endpoint Webhook trên server của bạn (ví dụ: https://oms.midomax.vn/api/shopee/webhook).Cơ chế: Khi có đơn hàng mới hoặc trạng thái đơn hàng thay đổi, Shopee sẽ chủ động "đẩy" (Push) một thông báo JSON về server của bạn.Xử lý: Server của bạn nhận dữ liệu và cập nhật vào Database của CRM ngay lập tức.
